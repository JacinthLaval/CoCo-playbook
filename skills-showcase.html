💡 What Is a CoCo Skill?

A skill is a reusable instruction set that lives inside the Cortex Code CLI. When you say "build me a FHIR-to-OMOP app", the CLI loads the matching skill — complete with architecture patterns, known pitfalls, tested code templates, and a gated workflow — and guides you through a production-quality build in minutes instead of days.

Skills are just Markdown files in ~/.snowflake/cortex/skills/. You can zip one up, push it to GitHub, or hand it to a teammate. Every hard-won lesson from a real build gets encoded once and reused forever.

The compounding effect: Each project we built produced one or more skills. Each new project consumed existing skills and produced new ones. By skill #7, new builds start 10× faster because they stand on the shoulders of every previous build.

SPCS SPA Auth

The authentication pattern that makes SPCS web apps actually work

Infrastructure

Documents the only working pattern for SPAs on Snowpark Container Services to query Snowflake data. The SPCS service token does not work with the SQL REST API — you must use a Python backend with snowflake-connector-python and the auto-injected OAuth token. This skill encodes 3 days of hard-won debugging into a 5-minute scaffold.

Browser → SPCS Ingress → nginx:8080 → SPA static files → /api/* proxy → Flask:8085 ↓ snowflake-connector-python auth: /snowflake/session/token authenticator: 'oauth'
What It Builds
nginx + Flask + React scaffold with working SPCS auth
Origin Project
FIMR Dashboard — first React SPA on SPCS
Reused By
OncoLook Admin, FHIR Ingestion Manager, TEFCA Consent Portal, Ambient Scribe
Critical Pitfall
NEVER set SNOWFLAKE_HOST or SNOWFLAKE_ACCOUNT env vars — SPCS auto-injects internal values

App Testing

Graduated testing — start small, validate, then scale with confidence

Quality Assurance

A three-level testing framework that prevents expensive failures. Level 1 proves the pipeline runs (1 record). Level 2 validates proportional output (10 records). A mandatory pause forces review before Level 3 runs at full scale. Extracted from the FHIR-to-OMOP test suite after we learned that skipping straight to 1,000 bundles wastes time and compute.

Level 1 (Smoke) 1 record → Does it run? ↓ Level 2 (Validate) 10 records → Are outputs proportional? ↓ ⚠ MANDATORY PAUSE → Review results, get approval ↓ Level 3 (Scale) Full data → Ground truth comparison (±10%)
What It Builds
Test script with --level flag, dedicated test schema, auto-cleanup
Origin Project
FHIR-to-OMOP Transformer — graduated bundle testing
Reused By
OncoLook inference pipeline, Ontology Stack Builder validation
Key Insight
The pause between Level 2 and 3 prevents 90% of expensive test failures

📦 How to Share Skills

Skills are portable. A skill is a directory with a SKILL.md file and optional reference docs. To share one:

1. Zip the skill directory: zip -r agent-guardrails.zip ~/.snowflake/cortex/skills/agent-guardrails/
2. Or push to GitHub: any CoCo user clones the repo into their ~/.snowflake/cortex/skills/ directory
3. The CLI auto-discovers it — just use the trigger phrases in conversation

No installation, no package manager, no configuration. Skills are Markdown files that encode expertise. Drop them in the folder and they work.

Skills Are How AI Expertise Compounds

Every hard build produces a skill. Every future build starts faster. This is the flywheel — not just building with AI, but teaching the AI how you build, so it gets better with every project.
